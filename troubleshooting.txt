1. /cmd_vel 메시지 타입 불일치 문제

터틀봇 - geometry_msgs/msg/TwistStamped을 받아야 함
Nav2 - geometry_msgs/msg/Twist, geometry_msgs/msg/TwistStamped 두 가지 메세지 타입 동시에 발행

burger.yaml에서 Nav2 관련 노드들의 enable_stamped_cmd_vel을 모두 true로 맞춤

2. Nav2 출력 토픽과 TurtleBot 입력 토픽이 다름

중간에 토픽 목록
/cmd_vel
/cmd_vel_nav
/cmd_vel_smoothed
/cmd_vel_teleop
/tb3/cmd_vel
/tb3/cmd_vel_smoothed

/controller_server → /cmd_vel_nav
/velocity_smoother → /cmd_vel_smoothed 이런 식으로 전역 토픽을 발행함
터틀봇은 /tb3 가 붙은 namespace 토픽을 받아야 함

burger.yaml에서 collision_monitor의 입력/출력을 namespace에 맞게 맞춤
cmd_vel_in_topic: "/cmd_vel_smoothed"
cmd_vel_out_topic: "/tb3/cmd_vel"
enable_stamped_cmd_vel: true

3. /tb3/cmd_vel이 아예 안 나오는 문제, TurtleBot 라이다 /tb3/scan이 안 나오던 문제

/cmd_vel_nav와 /cmd_vel_smoothed에는 값이 나왔는데, 최종 토픽인 /tb3/cmd_vel에는 아무것도 안 나오는 문제 발생 -> controller service, velocity smoother는 정상이고 최종 단계인 collision monitor에서 끊기는 문제 -> /tb3/scan을 구독하지만 터틀봇의 라이다 데이터가 정상적으로 출력되지 않음

LDS-01에서 LDS-02로 라이다 교체 후 ros2 topic echo /tb3/scan sensor_msgs/msg/LaserScan --qos-reliability best_effort --once 확인한 결과 scan 데이터 정상적으로 출력됨.

4. /tb3/scan frame_id가 잘못 붙는 문제

라이다 교체 후 /tb3/scan은 나오기 시작했지만, frame_id가 아래 처럼 출력
frame_id: tb3/tb3/base_scan

robot.launch.py에서 라이다 frame_id를 이미 tb3/base_scan으로 넣어둔 상태였는데, launch 실행 시 namespace도 tb3로 줘서 tb3이 중복으로 써짐.

turtlebot3_bringup/launch/robot.launch.py에서 라이다 frame_id를 다시 아래처럼 수정.
'frame_id': 'base_scan'

수정 후 /tb3/scan 확인 결과: frame_id: tb3/base_scan 정상적으로 출력!!~~

# 최종 요약
1. cmd_vel 타입 불일치
   → enable_stamped_cmd_vel: true로 해결

2. Nav2 출력 토픽과 TurtleBot 입력 토픽 불일치
   → collision_monitor 출력 토픽을 /tb3/cmd_vel로 수정

3. /tb3/cmd_vel이 안 나옴
   → collision_monitor 이후에서 끊기는 문제로 확인

4. collision_monitor가 /tb3/scan을 필요로 하는데 TurtleBot 라이다 scan이 안 나옴
   → 라이다 교체 후 /tb3/scan 정상 출력

5. /tb3/scan frame_id가 tb3/tb3/base_scan으로 잘못 나옴
   → robot.launch.py의 frame_id를 base_scan으로 수정

6. tb3/base_link → tb3/base_scan TF 확인
   → 정상 연결 확인

7. collision_monitor가 /tb3/cmd_vel을 정상 publish하기 시작
   → TurtleBot이 실제 이동 가능해짐

8. 현재는 목적지까지 가지만 주행이 불안정함
   → 이제는 Nav2 파라미터 튜닝 및 map→odom 정렬 단계
   
   토픽 타입 문제
→ namespace 토픽 연결 문제
→ 라이다 scan 미출력 문제
→ scan frame_id 중복 문제
→ collision_monitor 출력 문제
→ 주행 파라미터 튜닝 단계
